# AIO 输入框 AIOInput 组件规范

> **组件 ID**：`aio_input`  
> **大类**：操作  
> **变体数量**：3 种（I1默认态 / I2生成中态 / I3输入态）

---

## 1. 组件概述

AIO 输入框是用于 AI 对话场景的底部固定输入组件，集成了文字输入、语音输入功能。组件采用胶囊形设计，通过不同状态展示默认态、生成中态和输入态。

### 适用场景
- AI 助手对话界面
- 智能问答输入
- 语音转文字交互
- 聊天消息发送

---

## 2. 交互状态切换

AIO 输入框通过 **3 段式分段选择器** 在以下三种交互状态之间切换：

| 状态 | 编号 | 分段选择标签 | 说明 |
|------|------|-------------|------|
| 默认态 | I1 | 默认态 | 用户未输入时的初始状态，显示占位文字"发消息" |
| 生成中态 | I2 | 生成中态 | AI 正在生成回复时的状态，显示音波动画图标 |
| 输入态 | I3 | 输入态 | 用户正在输入文字时的状态，显示实际输入内容 |

### 分段选择器交互规则
- 分段选择器位于变体矩阵控件栏，用于切换展示不同交互状态
- 三个状态选项并列排列，当前激活项高亮显示
- 点击不同选项可切换对应的组件状态预览
- 分段选择器样式复用系统通用的 `mc-segmented-bar` 样式

## 7. 属性约束

1. **背景色**：页面背景色使用白色 `#FFFFFF`（`--bg-bottom`）
2. **定位**：组件固定在页面底部
3. **毛玻璃效果**：外层容器必须带 `backdrop-filter: blur(25px)` 以融合底部内容
4. **圆角规则**：仅 I2 生成中态使用 100px 胶囊圆角，其余状态为 24px
5. **阴影规则**：仅 I2 生成中态有阴影
6. **图标位置**：左侧图标在 I1 默认态时 top: 12px，在 I2/I3 时 top: 14px

## 10. 状态转换条件

> **注意**：分段选择器仅用于变体矩阵/组件构建器中的预览切换，不代表真实交互流转。以下为实际用户交互中的状态机：

| 当前状态 | 触发条件 | 目标状态 | 说明 |
|---------|---------|---------|------|
| I1 默认态 | 用户点击输入框区域 | I3 输入态 | 键盘弹起，输入框获得焦点 |
| I3 输入态 | 用户点击发送按钮 | I2 生成中态 | 消息发送，AI 开始生成回复 |
| I2 生成中态 | AI 生成完成 | I1 默认态 | 生成结束，输入框恢复默认 |
| I3 输入态 | 输入框失焦且内容为空 | I1 默认态 | 空内容时自动回到默认态 |
| I2 生成中态 | 用户点击停止按钮 | I1 默认态 | 中断 AI 生成，恢复默认态 |

## 11. I2 生成中态动画参数

| 属性 | 值 |
|------|------|

| 持续时长 | 1500ms |

| 缓动曲线 | ease-in-out |
| 图标路径 | CSS 实现（`width:2px; height:23px; background:var(--brand-standard)`） |
| 实现方式 | CSS `@keyframes` 或 SVG SMIL animation |

## 12. 与 Message 组件联动

AIO Input 与 Message 组件在 AIO 对话场景中紧密配合：

### 12.1 布局关系
- AIO Input **固定在页面底部**（`position: fixed; bottom: 0`）
- Message 消息列表在上方滚动，底部需留出 AIO Input 高度（72px）的 `padding-bottom` 避免内容遮挡
- 两者共享页面背景色 `--bg-select`（var(--bg-secondary)）

### 12.2 发送时序
```
I3(输入态) → 用户点击发送
  ├─ 1. 生成用户消息气泡（Message 主态）
  ├─ 2. AIO Input 切换到 I2(生成中态)
  ├─ 3. AI 流式输出对应客态气泡（Message 客态）
  └─ 4. 生成完成 → AIO Input 恢复 I1(默认态)
```

### 12.3 消息列表滚动规则
- 新消息到达时，列表自动滚动到底部
- 用户手动上滑查看历史时，不自动滚动（保持当前位置）
- I2 生成中态期间，新增的流式内容持续触发滚动到底部
