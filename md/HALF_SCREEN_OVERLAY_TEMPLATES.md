# 半屏浮层 · 场景模版库

> **文档定位**：本文档是 `HALF_SCREEN_OVERLAY_COMPONENT_SPEC.md` 的配套场景模版库，定义了常用半屏浮层场景的标准组合方式。  
> **使用规则**：执行半屏浮层设计任务时，**优先查阅本文档**，判断是否可直接使用或小幅修改某个模版，再按规范生成代码。如果实际任务或场景不满足任何模版，则基于 `HALF_SCREEN_OVERLAY_COMPONENT_SPEC.md` 规范约束，使用 Basic 1.0 设计组件库自行开展设计。所有模版均已通过 Basic 1.0 规范校验。  
> **前置必读**：使用模版前需读取 `HALF_SCREEN_OVERLAY_COMPONENT_SPEC.md`（浮层容器规范）和 `DIVIDER_SPACING_COMPONENT_SPEC.md`（间距规则）。

---

## 模版匹配机制

执行半屏浮层设计任务时，按**核心特征**判定最匹配的模版，而非依赖单一关键词：

| 模版 | 核心特征（需同时满足） | 典型场景 |
|------|----------------------|---------|
| **T-01** 图文新手引导 | ① 有**大面积图片/插画**展示 ② 有**简短文字说明**（非长文） ③ 底部有**操作按钮**引导下一步或完成 | 功能亮点引导、新特性介绍、onboarding |
| **T-02** 长文介绍 | ① **纯文字或图文混排**的长内容 ② 无操作按钮（纯阅读） | 帮助说明、变更公告、功能详细介绍 |
| **T-03** 权限授权确认 | ① 需要用户**主动确认/同意**某些条款或权限 ② 有**多条勾选项或权限列表** ③ 底部有**确认+拒绝**双按钮 | 权限申请、协议确认、隐私政策同意、账号授权 |
| **T-04** 二级设置列表 | ① 以**卡片式列表（Grouped List）**为主体 ② 各行点击后**跳转或切换开关**，无底部操作按钮 ③ 使用**二级导航栏**（带返回箭头） | 偏好设置、筛选条件、选项配置 |
| **T-05** 状态反馈 | ① 顶部有**大图标**（成功✓/失败✗/提示ℹ） ② 居中标题+说明文字 | 操作成功/失败、警告提示、完成确认 |
| **T-06** 输入表单 | ① 核心交互为**文本输入**（TextField） ② 需要**系统键盘**弹出 ③ 面板较小，收集简短信息 | 文本录入、搜索弹窗、表单填写 |

### 匹配决策流程

```
任务分析 → 提取核心交互意图
    ↓
按核心特征逐条比对 → 找到最匹配的模版
    ↓
完全匹配 → 直接使用
部分匹配 → 小幅修改复用
无匹配   → 参考模版自由组合
```

### 多特征冲突解决

当任务同时符合多个模版特征时，以**核心交互意图**为准：
- "需要用户确认权限的设置页面" → 核心是**确认权限** → **T-03**（非 T-04）
- "展示功能介绍的文字长文" → 核心是**长内容阅读** → **T-02**（非 T-01）
- "输入表单提交后的成功反馈" → **两个连续页面**，分别用 **T-06** + **T-05**

> **边界场景**：若任务不匹配以上任何模版特征，或需要超出模版范围的组合，半屏浮层支持在 `HALF_SCREEN_OVERLAY_COMPONENT_SPEC.md` 规范约束下，使用 Basic 1.0 各类组件自由搭配。

## 模版字段说明

### 组件堆叠标注规则

| 标注 | 含义 |
|------|------|
| 🔒 锁定 | 必须存在，变体不可更换，仅文案/图片内容可替换 |
| 🔄 可切换 | 必须存在，但可在括号内列出的变体范围内切换 |
| ➕ 可增加 | 可在此位置额外插入同类组件（数量≥0） |

间距格式：`↕ spacing-l（16px）— 规则：[规则描述]`，增减组件时查阅 `DIVIDER_SPACING_COMPONENT_SPEC.md` 选择矩阵推导。

### T-01 图文新手引导

> 适用于向新用户展示功能亮点，包含图片（支持轮播）+ 标题 + 正文 + 双按钮操作。

**浮层规格**：标准型 HSO-A · HS_NavBar A3（仅关闭） · 背景色 `#FFFFFF`

**组件堆叠**

```
┌─────────────────────────────────────────────────────┐
│  🔒 HS_NavBar A3（54px）
│
│  🔄 ImageBlock（428×320px）
│     可切换：单图有轮播（含底部指示器）↔ 单图无轮播
│
│  ↕ spacing-l（16px）— 规则：不同组件相邻，默认 16px
│
│  🔒 TextBlock H1 大标题（50px）
│
│  ↕ spacing-s（8px）— 规则：同类组件相邻
│
│  🔒 TextBlock H4 正文段落（至少 1 段，高度自适应）
│
│  ➕ 可增加：TextBlock H4 / H6 / H7
│     ↕ spacing-s（8px）— 规则：同类组件相邻
│
│  ↕ spacing-xl（24px）— 规则：不同功能模块之间
│     注：始终位于最后一个 TextBlock 与 ActionCombo 之间
│
│  🔄 ActionCombo（52px）
│     可切换：A1 双按钮（二级+一级）↔ A3 单按钮（一级满宽）↔ A4 单按钮（二级满宽）
│
│  ↕ spacing-xxl（32px）— 规则：最后组件 → 页面底部
│
│  🔒 HomeBar（34px）
└─────────────────────────────────────────────────────┘
```

**注意事项**

1. HS_NavBar A3 与 ImageBlock **层叠**（非间距关系），导航栏绝对定位于图片之上，不占内容流高度
2. TextBlock H4 至少保留 1 段，不可删除

Figma: `node-id=4133:57939`

### T-02 长文介绍（把手型）

> 适用于帮助说明、功能介绍、变更公告等纯文字或图文混排的长内容场景，支持上滑全屏阅读。

**浮层规格**：把手型 HSO-B · 背景色 `#FFFFFF`

**组件堆叠**

```
┌─────────────────────────────────────────────────────┐
│  🔒 把手条（16px）
│
│  ↕ spacing-l（16px）— 规则：HSO-B 把手条 → 内容区顶部固定间距
│
│  ➕ 可自由组合：TextBlock 任意变体
│     遵循 TEXT_BLOCK_COMPONENT_SPEC.md §2 Markdown 渲染规范
│     ↕ spacing-s（8px）— 规则：同类组件相邻
│
│  ➕ 可自由穿插：ImageBlock 嵌入变体（428×224px，含 px-16 py-12）
│     ↕ spacing-m（12px）— 规则：TextBlock 与 ImageBlock 之间（上下均适用）
│
│  🔒 HomeBar（34px）
└─────────────────────────────────────────────────────┘
```

**自由组合规则**

1. 文本块遵循 Markdown 渲染规范，标题层级递减（H1→H2→H3），不可跨级
2. ImageBlock 嵌入变体（px-16 py-12，图片实际高度 200px）可出现 0 次或多次
3. 无 ActionCombo，纯阅读场景

**注意事项**

1. 把手型支持上滑全屏，全屏态导航栏 L3 关闭必选，详见 `HALF_SCREEN_OVERLAY_COMPONENT_SPEC.md` §6.6

Figma: `node-id=4133:57504`

### T-03 权限授权确认

> 适用于应用权限申请、账号授权、多条协议勾选确认等场景。

**浮层规格**：标准型 HSO-A · HS_NavBar A4（左图标+应用名称，右全屏按钮） · 背景色 `#FFFFFF`

**组件堆叠**

```
┌─────────────────────────────────────────────────────┐
│  🔒 HS_NavBar A4（54px）
│
│  ↕ spacing-s（8px）— 规则：HS_NavBar → 内容组件
│
│  🔒 TextBlock H2 一级标题（36px）
│
│  ↕ spacing-xs（4px）— 规则：同组件之间·紧凑
│
│  🔒 TextBlock H6 摘要文本（40px）
│
│  ↕ spacing-l（16px）— 规则：不同组件相邻，默认 16px
│
│  🔄 Slot-1 内容区（见下方）
│
│  ↕ spacing-xl（24px）— 规则：不同功能模块之间
│
│  🔄 Slot-2 操作区（见下方）
│
│  ↕ spacing-xxl（32px）— 规则：最后组件 → 页面底部
│
│  🔒 HomeBar（34px）
└─────────────────────────────────────────────────────┘
```

---

**🔄 Slot-1 内容区**

插入 **同一种** Plain List 子组件，重复 1 条或多条。从以下三种组合中选一种填入：

| 选项 | 子组件规格 | 单条高度 | 条目间隔 | 适用场景 |
|------|-----------|--------|---------|---------|
| **S1-A**（默认） | Plain List · S1 · L0 · C1 · R0（多选+标题）| 44px | 分割线 | 多条权限/协议勾选，无辅助说明 |
| **S1-B** | Plain List · S0 · L5 · C2 · R0（大头像+标题+描述，无跳转） | 72px | 分割线 | 展示应用/账号信息 |
| **S1-C** | Plain List · S0 · L0 · C2 · R0（标题+描述，无跳转） | 72px | 分割线 | 每条附辅助说明 |

> **约束**：同一 Slot-1 内只能使用同一种子组件规格，不可混排。行间用分割线（0px 间距），最少 1 条。

---

**🔄 Slot-2 操作区**

插入 **ActionCombo** 组件，支持两种配置：

| 配置 | 构成 | 高度 | 适用场景 |
|------|------|------|---------|
| **仅操作行 A** | ActionCombo A 类（A1-A8 选一） | 52px（A1-A4）/ 72px（A5）/ 48px（A6-A7）/ 72px（A8） | 简单确认/拒绝，无附加说明 |
| **操作行 A + 辅助操作行 B** | ActionCombo A 类 + B 类各选一，B 行紧跟 A 行下方 | A 高度 + 36px | 需要协议勾选、辅助说明或附加文字链 |

> **约束**：操作行 A 必选，辅助操作行 B 可选；B 行不可单独使用（必须跟随 A 行）。

**典型搭配示例**

| 场景 | Slot-1 | Slot-2 |
|------|--------|--------|
| 隐私协议勾选确认 | S1-A × N | A1（双按钮）+ B1（居左勾选） |
| 应用账号授权 | S1-B × N | A1（双按钮） |
| 权限申请（附说明文字链） | S1-C × N | A1（双按钮）+ B2（居左辅助信息） |

**注意事项**

1. H2 + H6 为**紧凑标题组**，间距固定 spacing-xs（4px），不可改为 B2
2. Slot-1 行间用**分割线而非间距组件**
3. S1-A 使用 Plain List 多选态（S1），点击条目切换勾选状态；S1-B / S1-C 使用默认态（S0），无勾选交互

Figma: `4140:58745`（S1-A 示例）/ `4152:4048`（S1-B 示例）/ `4152:47172`（S1-C 参考，R0 修正）

### T-04 二级设置列表

> 适用于二级设置项浏览与跳转，以卡片式列表（Grouped List）组合为核心，支持多组设置项分组展示。

**浮层规格**：标准型 HSO-A · HS_NavBar B 类（三选一，见下方） · 背景色 `var(--bg-secondary)`（含 Grouped List 必须用灰底）

**组件堆叠**

```
┌─────────────────────────────────────────────────────┐
│  🔄 HS_NavBar B 类（54px，三选一：B1 / B2 / B3）
│
│  ↕ spacing-s（8px）— 规则：HS_NavBar → 内容组件
│
│  ➕ 可增加：Grouped List 表单组合（Combo2 / Combo4 / Combo5）
│     ↕ spacing-m（12px）— 规则：同类卡片容器相邻
│
│  ↕ spacing-xxl（32px）— 规则：最后组件 → 页面底部
│
│  🔒 HomeBar（34px）
└─────────────────────────────────────────────────────┘
```

**🔄 HS_NavBar B 类三选一**

| 变体 | 右侧 | 适用场景 |
|------|------|---------|
| B1 | 文字操作 | 需要右上角操作（如"编辑"） |
| B2 | 关闭按钮 | 需要关闭整个浮层 |
| B3 | 无 | 最简导航，无额外操作 |

**➕ 表单组合配置**

Grouped List 遵循 `GROUPED_LIST_COMPONENT_SPEC.md`，Figma 示例为 **Combo5（分组标题 + 多行 L3·R1）**，其他合规组合（Combo2/4）均可使用。

**注意事项**

1. HS_NavBar B 类左侧返回箭头为**返回上一级**（非关闭浮层），仅 B2 的关闭按钮关闭整个浮层
2. 无 ActionCombo，设置项即点即生效

### T-05 状态反馈

> 适用于操作结果反馈（成功/失败/警告/提示），顶部固定状态图标+标题+正文，下方可配操作按钮、宫格、列表等。

**浮层规格**：标准型 HSO-A · HS_NavBar A3（仅关闭） · 背景色 `#FFFFFF`

**组件堆叠**

```
┌─────────────────────────────────────────────────────┐
│  🔒 HS_NavBar A3（54px）
│
│  🔒 状态图标（64×64px，居中）
│     可切换图标（限以下三种）：
│     · caution_filled.svg — var(--accent-red)（错误/失败）
│     · tick_filled.svg    — var(--accent-green)（成功/完成）
│     · info_filled.svg    — var(--text-secondary)（提示/中性）
│     路径：icons/QUI_24_icons/，渲染尺寸 64×64px（CSS scale）
│
│  ↕ spacing-l（16px）— 规则：不同组件相邻，默认 16px
│
│  🔒 TextBlock C1 大标题（居中，50px）
│
│  ↕ spacing-s（8px）— 规则：同类组件相邻
│
│  🔒 TextBlock C4 正文段落（居中，至少 1 段，高度自适应）
│     支持内联 Textlink
│
│  ↕ spacing-xl（24px）— 规则：不同功能模块之间
│
│  🔄 Slot-1 内容区（可选，见下方）
│
│  ↕ spacing-xl（24px）— 规则：不同功能模块之间
│     注：Slot-1 为空时省略此间距
│
│  🔄 Slot-2 操作区（可选，见下方）
│
│  ↕ spacing-xxl（32px）— 规则：最后组件 → 页面底部
│
│  🔒 HomeBar（34px）
└─────────────────────────────────────────────────────┘
```

---

**🔄 Slot-1 内容区（可选）**

| 选项 | 组件 | 说明 |
|------|------|------|
| **不配置**（默认） | 无 | 纯文字反馈，面板最小 |
| **宫格** | Grid 任意变体；占位为图片（横版/竖版/正方形）或图标（圆形），标题文字可选 | 反馈后的快捷操作入口 |
| **列表** | Plain List 任意合规 L×R 组合（S0） | 反馈后的相关内容推荐 |

> **约束**：宫格可不显示文案（仅图标），渲染时将文案设为空字符串。

---

**🔄 Slot-2 操作区（可选）**

| 选项 | 组件 | 高度 | 适用场景 |
|------|------|------|---------|
| **不配置**（默认） | 无 | — | 纯展示反馈，无需用户操作 |
| **操作行 A** | ActionCombo A 类（A1-A8 选一） | 52px（A1-A4）等 | 需要用户主动操作（如"再试一次"/"去设置"） |

> **约束**：Slot-2 无辅助操作行 B（状态反馈场景不需要协议勾选类辅助操作）。

**典型搭配示例**

| 场景 | Slot-1 | Slot-2 |
|------|--------|--------|
| 操作成功，无后续 | 不配置 | 不配置 |
| 操作成功，推荐后续操作 | 宫格 | 不配置 |
| 操作失败，可重试 | 不配置 | A3（单一级按钮） |
| 支付成功，推荐相关内容 | 列表 | 不配置 |

**注意事项**

1. HS_NavBar A3 与状态图标**层叠**（非间距关系），导航栏绝对定位于内容之上，不占内容流高度
2. 状态图标**只能**使用 `caution_filled` / `tick_filled` / `info_filled` 三种，不可使用其他图标
3. 文本块固定使用**居中对齐**（C 类），与 T-01 的居左不同
4. Slot-1 与 Slot-2 均为空时，正文段落直接接 B6 → HomeBar

Figma: `node-id=4165:46156`

### T-06 输入表单

> 适用于简单文本输入场景，面板较小，底部弹出系统键盘，输入框数量可增加，多个单行输入框按类别组合为卡片。

**浮层规格**：标准型 HSO-A · HS_NavBar A1（标题+关闭） · 背景色 `#FFFFFF`

**组件堆叠**

```
┌─────────────────────────────────────────────────────┐
│  🔒 HS_NavBar A1（54px）
│
│  ↕ spacing-s（8px）— 规则：HS_NavBar → 内容组件
│
│  🔄 TextField（56px，可增加，见下方组合规则）
│     可切换类型：A 单行无标题 ↔ B 单行有标题 ↔ C 电话号码 ↔ D 多行文本
│     默认显示默认态（状态 1）
│
│  ↕ spacing-xxl（32px）— 规则：最后组件 → 页面底部
│
│  🔒 HomeBar（34px）
└─────────────────────────────────────────────────────┘
```

**➕ TextField 增减与组合规则**

单个 TextField 直接放置，多个时须遵循以下卡片组合规范（参照 Grouped List 组合逻辑）：

| 规则 | 说明 |
|------|------|
| **同类单行输入组合** | 同一类型（A/B/C）的多个单行 TextField 合并为一个卡片容器，12px 圆角，背景白色 |
| **卡片内分割线** | 同卡片内相邻行之间显示 0.5px 分割线（`var(--border-default)`），左右缩进 16px |
| **卡片间间距** | 不同卡片之间 ↕ spacing-m（12px）— 规则：同类卡片容器相邻 |
| **多行 TextField（D 类）** | D 类独立放置，不与其他类型合并为卡片，与相邻卡片间距 ↕ spacing-s（8px） |
| **最多行数** | 卡片内行数不限，但面板高度最大 720px，超出可滚动 |

**注意事项**

1. 面板弹出后系统键盘自动顶起，面板实际可见高度 = 面板高度 - 键盘高度（约 333px）
2. TextField 展示态为默认态（状态 1），代码实现时需绑定 focus/blur/input 事件切换状态

Figma: `node-id=4166:3204`
