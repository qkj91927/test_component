# 卡片 Card · 组件设计规范

> **组件 ID**：`card`  
> **大类**：数据  
> **变体数量**：10 种子组件（C1 ~ C10）

---

## 1. 组件概述

卡片组件是一种承载结构化内容的圆角容器，用于在信息流、详情页等场景中展示多种组合形式的内容块。每张卡片由多个区块（头像行、图片区、标题区、辅助信息行、列表行、按钮区、Markdown 内容区等）自由组合而成，统一使用 24px 圆角白色容器，两侧缩进 16px（内容宽度 396px）。

## 3. 全局结构参数

### 3.1 卡片容器

| 属性 | 值 | 说明 |
|------|------|------|
| 外层宽度 | 428px | 设备宽度 |
| 内容宽度 | 396px | 左右各缩进 16px |
| 圆角 | 24px | 统一圆角 |
| 背景色 | `#FFFFFF` (`--bg-bottom`) | 白色容器 |
| 溢出裁剪 | `overflow: hidden` | 图片撑满不超出 |

### 3.2 页面背景色约束

卡片组件为圆角白色容器，**页面背景色必须使用** `var(--bg-secondary)`，与卡片式列表(Grouped List)规则一致，白色卡片需要灰色底形成视觉层级分离。

## 5. 属性约束

1. **圆角统一**：所有卡片容器圆角必须为 24px，不可修改
2. **宽度固定**：卡片内容宽度 396px，两侧 16px 边距，不可调整
3. **图片比例**：短图 180px、长图 320px 为固定高度，宽度 396px 撑满
4. **头像行分割线**：仅底部展示分割线，无头像行时无此分割线
5. **九宫格布局**：1大+4小固定布局，大图必须在左侧，不支持其他排列
6. **辅助信息行**：始终在卡片底部，高度固定 40px
7. **大按钮**：大按钮类型仅为二级样式（灰色背景），不支持一级（蓝色）
8. **箭头行**：每行顶部必须有分割线，箭头为右侧 chevron_right 图标
9. **背景色**：页面使用卡片组件时，背景色必须为 `var(--bg-secondary)`
10. **C10 高度自适应**：C10 卡片高度根据 Markdown 内容自适应，最大不超过 840px

## 6. C10 Markdown 内容规范

C10 卡片专用于承载 Markdown 渲染后的文本块内容，适用于 AI 生成内容展示、富文本详情页等场景。

### 6.1 可用文本块类型

C10 Markdown 内容区仅支持以下居左对齐（H 类）文本块子组件：

| 文本块编号 | 名称 | 字号 | 字重 | 行高 | 适用场景 |
|-----------|------|------|------|------|----------|
| H2 | 一级标题 | 22px | 500 | 28px | 卡片内章节标题 |
| H3 | 二级标题 | 18px | 500 | 26px | 卡片内子章节标题 |
| H4 | 正文段落 | 17px | 400 | 24px | 主要内容文本 |
| H5 | 前缀正文 | 17px | 400 | 24px | 有序/无序列表项 |
| H6 | 摘要文本 | 14px | 400 | 20px | 引用块、注释 |
| H7 | 提示文本 | 12px | 400 | 20px | 脚注、辅助说明 |

> **注意**：C10 卡片内**不使用 H1 大标题**，因为卡片头像行已提供标题，内容区从章节标题（H2）开始。

### 6.2 Markdown 语法映射

| Markdown 语法 | 映射的文本块 | 说明 |
|---------------|-------------|------|
| `## 二级标题` | H2 一级标题 | 卡片内顶级章节标题 |
| `### 三级标题` | H3 二级标题 | 卡片内子章节标题 |
| 普通段落 | H4 正文段落 | 主要内容 |
| `1. 有序列表` / `- 无序列表` | H5 前缀正文 | 列表项 |
| `> 引用` | H6 摘要文本 | 引用块 |
| 脚注 / `<small>` | H7 提示文本 | 辅助信息 |
| `[链接](url)` | Textlink | 蓝色链接文字 `#214CA5` |

### 6.3 文本块间距规范（卡片内）

卡片内文本块间距遵循紧凑原则，相比页面级文本块略有收紧：

#### 6.3.1 标题后间距

| 上方元素 | 下方元素 | 间距值 |
|----------|----------|--------|
| H2 一级标题 | H3 二级标题 | **12px** |
| H2 一级标题 | H4 正文段落 | **12px** |
| H3 二级标题 | H4 正文段落 | **8px** |
| H3 二级标题 | H5 前缀正文 | **8px** |

#### 6.3.2 正文/列表间距

| 上方元素 | 下方元素 | 间距值 |
|----------|----------|--------|
| H4 正文段落 | H4 正文段落 | **8px** |
| H4 正文段落 | H5 前缀正文 | **8px** |
| H5 前缀正文 | H5 前缀正文 | **4px** |
| H5 前缀正文 | H4 正文段落 | **8px** |

#### 6.3.3 辅助文本间距

| 上方元素 | 下方元素 | 间距值 |
|----------|----------|--------|
| H4 正文段落 | H6 摘要文本 | **12px** |
| H4 正文段落 | H7 提示文本 | **12px** |
| H6 摘要文本 | H7 提示文本 | **8px** |

#### 6.3.4 章节分隔间距

| 场景 | 间距值 |
|------|--------|
| 新章节（H2 一级标题前） | **20px** |
| 新子章节（H3 二级标题前） | **16px** |

### 6.4 Textlink 链接色

在 H4/H5/H6/H7 文本中可嵌入 Textlink 链接色文字：

| 属性 | 值 |
|------|------|
| 颜色 | `#214CA5` (`var(--text-link)`) |
| 字号 | 继承父容器 |
| 下划线 | 无 |
| 光标 | pointer |

### 6.5 推荐内容模板

**模板 A：AI 回答卡片**
```
[头像行：AI 助手标题]
──────────────────────
H2 一级标题（回答主题）
  └─ H4 正文段落（解释说明）
  └─ H4 正文段落

H3 二级标题（步骤/要点）
  └─ H5 前缀正文（1. 第一步）
  └─ H5 前缀正文（2. 第二步）
  └─ H5 前缀正文（3. 第三步）

H7 提示文本（了解更多：[帮助链接]）
```

**模板 B：内容详情卡片**
```
[头像行：作者/来源标题]
──────────────────────
H4 正文段落（正文内容）
H4 正文段落

H6 摘要文本（> 重点引用内容）

H4 正文段落（继续正文）
H7 提示文本（发布时间 · 阅读量）
```

**模板 C：说明/帮助卡片**
```
[头像行：功能名称]
──────────────────────
H4 正文段落（功能简介）

H3 二级标题（使用方法）
  └─ H5 前缀正文（• 方法一）
  └─ H5 前缀正文（• 方法二）

H3 二级标题（注意事项）
  └─ H4 正文段落（注意说明）

H7 提示文本（如有疑问，请[联系客服]）
```

### 6.6 禁止的组合

| 禁止模式 | 原因 |
|----------|------|
| H1 大标题 | 卡片头像行已提供标题，不可再用 H1 |
| H7 → H4 正文 | 提示文本应为末尾，后面不应有正文 |
| 连续多个 H2 | 同级标题不可连续，中间需有内容 |
| 居中对齐（C 类）| C10 卡片仅支持居左排版 |

## 11. C10 组件嵌套关系

C1-C9 为固定结构卡片，内部区块已预定义。C10 为**开放插槽卡片**（头像行固定 + 内容区可自由组合），以下定义 C10 内容区支持嵌套的子组件类型。

### 11.1 可嵌套 — 内容类

| 子组件 | 说明 |
|--------|------|
| TextBlock（文本块） | Markdown 渲染核心，使用 H2-H7 居左子组件 |
| ImageBlock（图片块） | 通栏/缩进图片，宽度受限于内容区 364px |
| List（通栏列表） | 卡片内列表项（如设置项、选项列表） |
| Grid（九宫格） | 卡片内多图展示 |
| DividerSpacing（间距/分割线） | 内容区区块间的分隔 |

### 11.2 可嵌套 — 操作类

#### Button vs ActionCombo 选择规则

| 场景 | 使用组件 | 说明 |
|------|---------|------|
| 仅需一个操作按钮 | **Button** | 单独使用，如"立即体验""查看详情" |
| 需要多个按钮组合 | **ActionCombo** | 仅 A1-A5 变体支持卡片，自带按钮排列和间距规范（如确认+取消） |

#### Button 尺寸选择规则

| 尺寸 | 适用场景 | 对齐方式 |
|------|---------|---------|
| 大按钮（宽度自适应内容区 364px × 52px） | 卡片内主要操作（如"立即体验""查看详情"） | 水平居中，宽度撑满内容区 |
| 中按钮（宽度自适应内容区 × 44px） | 卡片内次要操作 | 水平居中，宽度撑满内容区 |
| 小按钮（72×36px） | 内容行右侧附属操作（如"关注""查看"） | 右对齐，跟随所在行布局 |

> **对齐规则**：底部操作区（大/中按钮、ActionCombo）水平居中，宽度自适应卡片内容区（364px）；行内操作（小按钮）右对齐，跟随所在行布局。Mini 按钮不适用于卡片场景。

### 11.3 禁止嵌套

| 类型 | 原因 |
|------|------|
| Card（卡片） | 禁止卡片嵌套卡片 |
| Grouped List（卡片式列表） | 自带圆角白底容器，与 C10 白底冲突 |
| Dialog / ActionSheet / HalfScreenOverlay / Menu | 模态组件，独立于卡片 |
| NavBar / HS_NavBar | 导航组件，不嵌入卡片 |
| Message / AIOInput | AIO 场景专用组件 |
| Search / DataFilter | 页面级组件，不适合卡片内 |
| Textfield（输入框） | 卡片为展示型容器，不承载表单输入 |

### 11.4 嵌套规则

1. **宽度约束**：子组件宽度受限于 C10 内容区宽度 **364px**（396px - 左右各 16px 内边距）
2. **间距规则**：子组件之间间距遵循 `DIVIDER_SPACING_COMPONENT_SPEC.md` 规范
3. **操作位置**：Button / ActionCombo 建议放在内容区底部
4. **高度限制**：内容区最大高度 **784px**（840 - 56px 头像行），超出启用 `overflow-y: auto`
5. **扩展原则**：未来新增的**非模态、非导航、非表单输入**类展示/操作组件，均可作为 C10 子组件候选
