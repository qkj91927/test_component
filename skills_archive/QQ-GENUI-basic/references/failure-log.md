# QQ-GENUI Basic Failure Log 模板

> 用途：本文件是项目根目录 `failure-log.md` 的初始化模板与填写规范。首次使用 `QQ-GENUI-basic` 时，若项目根目录不存在 `failure-log.md`，应立即在项目根目录创建该文件，并可直接复制本模板内容作为初始内容。
>
> 运行规则：
> 1. 每次生成或调整界面前，先读取项目根目录 `failure-log.md`
> 2. 每次发生失败、返工、重试或用户指出问题后，立即向项目根目录 `failure-log.md` 写入一条记录
> 3. 最终交付前，补全对应记录中的解决方式、预防规则、关联组件/规范与复发说明

## 写入时机

满足以下任一情况时，必须写入项目根目录 `failure-log.md`：

- 首次使用本 skill，且项目根目录尚未存在 `failure-log.md`
- 任一 Gate 执行失败，需要停下修复并重试
- 用户指出问题，要求返工、重试或“再改一版”
- 严格回归中发现不符合项，需要进入下一轮修复
- 输出违反 QQ_GenUI 规范（组件细则、结构语义、版式细节、图标本地化、交互行为）

## 追加规则

- 运行态日志文件位于**项目根目录**：`failure-log.md`
- 每次新增记录追加到文件顶部，确保最近问题最先被读取
- 记录必须具体到“错误现象 → 根因 → 当前修复动作 → 下次预防规则”
- 不写空泛总结，如“细心一点”“下次注意”
- 若同类错误再次发生，在旧条目下补充“复发说明”，或新增一条更具体的记录
- 若首次创建项目根目录 `failure-log.md`，可直接复制下方“初始化模板”内容

## 初始化模板

```md
# Failure Log

> 由 `QQ-GENUI-basic` 在首次使用时创建。每次生成或调整前先读；每次失败、返工或重试后必写。

## 日志条目

> 暂无记录。
```

## 条目模板

```md
## [日期] 标题
- **任务类型**: 页面生成 / 局部调整 / 合规修复 / 其它
- **触发场景**: 用户原始需求摘要
- **触发节点**: Gate 失败 / 用户反馈重试 / 回归修复 / 其它
- **失败表现**: 首次输出哪里失败，或用户为什么需要反复调整
- **根因分析**: 漏读了什么、误判了什么、哪个 Gate 执行不到位
- **当前修复动作**: 本轮准备如何修复或已经完成了什么修复
- **最终解决方式**: 最终如何修复（若当前尚未完成，可先写“处理中”）
- **预防规则**: 下次执行前必须新增的检查动作
- **关联组件/规范**: 涉及的组件名、SPEC 文件、Token、图标或布局规则
- **复发说明**: 无 / 第 N 次发生，和历史哪条类似
```
